# macOS 版 実行ファイルの使い方（エンドユーザー向け）

このドキュメントでは、macOS でビルド済みの Meeting Mojiokoshi アプリ（.app バンドル）を実行する方法を説明します。

## 入手方法
- GitHub Releases から最新の `MeetingMojiokoshi-*-macos-*.zip` をダウンロード（Apple Silicon または Intel 版を選択）。
- またはソースから自分でビルドした `dist/MeetingMojiokoshi.app` を使用。

## 実行手順

1. ZIP ファイルを任意の場所に展開します。
   - 推奨: `~/Applications/` フォルダに移動（またはデスクトップ/ダウンロードフォルダ）。

2. 展開された `MeetingMojiokoshi.app` をダブルクリック（または右クリック → 開く）。

3. **初回起動時のセキュリティ警告**:
   - macOS の Gatekeeper により「"MeetingMojiokoshi.app" は開発元が未確認のため開けません」というダイアログが出ます。
   - 右クリック → 「開く」を選択 → 「開く」をクリックして許可してください。
   - 一度許可すると、以後通常のダブルクリックで起動できます。

4. アプリが起動したら、通常通り使用します。
   - 出力先を選択
   - 会議音声デバイス / マイクを選択
   - モデル準備 → 録音開始 → 停止して文字起こし

## System Tray（タスクトレイ）機能

アプリをバックグラウンドで動作させたい場合に便利です。

- メインウィンドウを閉じると、自動的にシステムトレイ（メニューバー右側）に最小化されます。アプリは終了せず、録音を継続できます。
- トレイアイコンをクリック（または右クリック）するとメニューが表示されます：
  - 「ウィンドウを開く」：メイン画面を表示
  - 「録音開始」：録音を開始（状態に応じて有効/無効）
  - 「停止して文字起こし」：録音を停止して文字起こし（状態に応じて有効/無効）
  - 「終了」：アプリを完全に終了
- トレイアイコンは常に表示されます（ウィンドウを開いていてもOK）。
- 録音中はトレイから素早く操作可能で、デスクトップをすっきり保てます。

**注意**: 現在の配布版は通常の GUI アプリとして起動するため、Dock アイコンは表示されます。ウィンドウを閉じるとメニューバー側から再表示できます。

## 初回設定のポイント
- モデル準備（初回はインターネット接続が必要です。tiny または base モデル推奨）。
- macOS の場合、BlackHole などの仮想オーディオデバイスを「会議音声」として選択する必要があります（詳細は同梱 README.md 参照）。

## 注意点
- 録音には参加者の同意を必ず得てください。
- アプリは単一の .app バンドルなので、ゴミ箱に捨てるだけでアンインストールできます。
- Apple Silicon（M1/M2/M3...）と Intel Mac では別々のバイナリを使用してください。間違ったアーキテクチャのものは起動しません。

詳細は同梱の README.md を参照してください。
